# React + Vite Frontend

## Install

```powershell
cd frontend
npm install
```

## Run in development

```powershell
npm run dev
```

## Optional Firebase analytics config (local only)

Create a local config file (gitignored):

```powershell
Copy-Item src\firebase\firebase.config.local.example.js src\firebase\firebase.config.local.js
```

Then edit `src/firebase/firebase.config.local.js` with your Firebase project values.

Notes:

- `src/firebase/firebase.config.local.js` is ignored by git.
- `src/firebase/firebase.config.generated.js` is auto-generated by npm scripts and also ignored by git.
- If local config is missing, Firebase initialization is skipped automatically.

By default:

- requests to `/ml-api/*` are proxied to `http://localhost:8000/*` (FastAPI ML API)
- requests to `/crud-api/*` are proxied to `http://localhost:8080/*` (Spring CRUD API)

Local safety behavior:

- when the app is opened on `localhost` / `127.0.0.1`, frontend calls are forced to `/ml-api` and `/crud-api`
- this avoids accidentally using production API URLs from `.env.production` during local runs

If needed, override API bases directly:

```powershell
$env:VITE_ML_API_BASE_URL="http://localhost:8000"
$env:VITE_CRUD_API_BASE_URL="http://localhost:8080"
npm run dev
```
